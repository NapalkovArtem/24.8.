#Сценарий. Оплата заказа. Оплата наличными

##Цель: *Предоставить пользователю возможность выбора оплаты наличными с указанием суммы сдачи.*

###**Основной сценарий**:

1. Пользователь выбирает тип оплаты “Оплата наличными“
2. Система меняет форму заказа и появляется дополнительное поле “Нужна сдача“ с типом булево.
3. Пользователь выбирает пункт “Истина“
4. Система выводит дополнительное поле “Укажите сумму, с которой нужна сдача“
5. Пользователь указывает сумму сдачи
6. Система делает доступной кнопку “Заказать“

####**Альтернативный сценарий**:

1. Пользователь выбирает тип оплаты “Оплата наличными“
2. Система меняет форму заказа и появляется дополнительное поле “Нужна сдача“ с типом булево.
3. Пользователь выбирает пункт “Ложь“
4. Система делает доступной кнопку “Заказать“

#####**Сценарий исключения**:

* Пользователь выбирает тип оплаты “Оплата наличными“
* Система меняет форму заказа и появляется дополнительное поле “Нужна сдача“ с типом булево.
* Пользователь выбирает пункт “Истина“
* Система выводит дополнительное поле “Укажите сумму, с которой нужна сдача“
* Пользователь указывает некорректную сумму(например, меньше чем сумма заказа)
* Система выдает ошибку “Неверно указана сумма. Сумма заказа превышает номинал суммы, которую Вы указали для сдачи“.
